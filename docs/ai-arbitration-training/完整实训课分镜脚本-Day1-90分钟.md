# 完整实训课分镜脚本 — Day 1 第一节

> **课名**：《90 分钟从零跑通 AI 劳动仲裁辅助系统》
> **对象**：第一次接触本项目的大学生（5~8 人小组 / 建议 2~4 个小组同步上课）
> **目标**：下课时，每位学生都能在自己电脑上发起一次"AI 法律问答"
> **时长**：90 分钟（含 10 分钟休息）
> **课前物料**：投影、白板、马克笔、A4 草稿纸、便利贴、WiFi

---

## 课前 30 分钟（讲师提前到）

> 这是讲师自己要做的事，不上课。

- [ ] 9:00 到教室，开投影，连接 HDMI / Type-C
- [ ] 打开 VS Code，分屏左 Shell 右 Editor
- [ ] 准备好的页面（提前在浏览器开 4 个标签页）：
  - 标签 1：当前骨架项目 仓库 `https://gitee.com/fastapiadmin/当前骨架项目`
  - 标签 2：本地 `http://127.0.0.1:8000/docs`（提前把项目跑起来）
  - 标签 3：OpenAI 平台 `https://platform.openai.com/`
  - 标签 4：uv 安装文档 `https://docs.astral.sh/uv/`
- [ ] 桌面放 4 个文件：`README.md` / `CLAUDE.md` / `项目难点总结.md` / `pyproject.toml`
- [ ] 准备一个共享 `.env`（把 `OPENAI_API_KEY=sk-xxx` 写在纸上，**不要在大屏幕上展示完整 key**）
- [ ] 白板画好分组表（按 4 人一组画好 4 个方格）

## 课程脚本

### Part 0：进门暖场（00:00 - 05:00）

#### 00:00 讲师推门进教室

**【动作】**
- 推门进来，背双肩包，手里拿激光笔
- 走到讲台中央，面朝学生，扫视一圈
- 把包放下，打开投影（按遥控器/笔记本快捷键 `Win+P`）

**【台词·逐字稿】**

> "同学们好，我是带你们做这个项目的讲师。不用自我介绍，反正接下来 4 天你们会恨死我，因为我会让你们装环境、改 bug、写到凌晨两点。"
>
> "开玩笑的。我们今天目标很明确：90 分钟之后，我要让你们每个人电脑上都有一个能跟你聊劳动法的 AI。能不能做到？"

**【互动】**
- 等学生喊"能" / 沉默
- 不管哪种都说："行，记住这句话，90 分钟后我会来检查。"

#### 02:00 设备检查

**【动作】**
- 走到学生中间，俯身看一两个学生的电脑
- 问："带电脑了吗？""电源插头带了没？""有没有没装 Python 的？举个手。"

**【台词】**

> "如果你电脑是今天第一次带过来，或者开机比老奶奶还慢，趁现在先去倒杯水、上个厕所。接下来的 90 分钟你不会有空。"

#### 03:00 课前调研

**【动作】**
- 走回讲台，拿起马克笔
- 在白板上画三列

**【台词】**

> "在开课之前，我想知道三件事，你诚实回答，不记名：
> 一、之前写过 Python 吗？会的举右手。
> 二、用过 Git 吗？会的举左手。
> 三、听过 RAG 这个词吗？会的站起来。
>
> 没关系，站着的不用坐下，蹲着就行。"

**【互动】**
- 看学生反应，记下比例
- 自嘲："我看到 RAG 那列基本是空的，这就是我们 4 天要学的东西。"

#### 05:00 切到 Part 1

### Part 1：项目震撼演示（05:00 - 15:00）

#### 05:00 切换到浏览器标签 2

**【动作】**

**【台词】**

> "RAG 是啥？等会儿 5 分钟讲完。Embeddings 是啥？也是 5 分钟。这两个搞懂，剩下 4 天就是写业务代码。"

### Part 2：团队组建（15:00 - 22:00）

#### 15:00 分组

**【动作】**
- 走回白板前
- 在白板上提前画好的方格里填名字

**【台词】**

> "现在分组。4 个人一组，按座位左右随便凑。凑不齐就 3 个或 5 个，**不要单飞**——一个人装环境出问题，没人救。"

#### 17:00 起组名 + 选角色

**【动作】**
- 给每个组发 3 张便利贴
- 让他们在便利贴上写：组名 / 组长名字 / "环境装不上"的求救暗号

**【台词】**

> "给你们 3 分钟。三件事：
> **1. 起个组名**（越土越好，叫"答辩不输队"的优先）
> **2. 选个组长**（最熟电脑的那个，不是成绩最好的）
> **3. 定一个求救暗号**——比如敲桌子三下。装环境装崩了别硬撑，敲暗号，全组过去看。"

#### 20:00 角色分配

**【动作】**
- 走到学生中间，听各组讨论
- 顺便纠正："组长不是大爷，是兜底的那个。装环境出错他先上。"

**【台词·追加】**

> "组里再分个工：
> - **操作手**：负责敲命令（一般就是组长）
> - **观察手**：盯着屏幕看错误信息，大声念出来
> - **记录手**：把踩坑写到便利贴上，等下贴在白板
> - **第四个人**……"（笑）"第四个人负责给大家买水。"

### Part 3：环境搭建实操（22:00 - 50:00）

#### 22:00 切到 Part 3 屏幕

**【动作】**
- 切到 VS Code 终端
- 把光标放大，字体调到 18

**【台词】**

> "好，戏演完了，开始干活。我现在带着大家一行一行敲。**不要复制粘贴**，手敲有手感。"

#### 23:00 第 1 步：装 uv

**【动作】**
- 终端输入命令（边输边念）

**【台词 + 屏幕同步】**

```powershell
# macOS / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh

# Windows PowerShell
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"
```

> "这段命令是装一个叫 **uv** 的工具，**这个项目强制用它**，pip 不好使，别问为什么。"

#### 25:00 验证 uv

**【动作】**
- 重开终端（或 `source ~/.cargo/env`）
- 敲 `uv --version`

**【台词】**

> "看到版本号说明装好了。看到 'command not found' 举手。"

**【救场话术】**
- 如果没人举手：跳过
- 如果有人举手：走过去重启终端 / 检查 PATH

#### 28:00 第 2 步：克隆代码

**【动作】**
- 切到浏览器标签 1，复制仓库地址
- 回到终端

**【台词 + 屏幕同步】**

```bash
git clone https://gitee.com/fastapiadmin/当前骨架项目.git
cd <项目目录>
```

> "如果你 git 没装，会报 'git 不是内部命令'，举手。"

**【互动】**
- 等 30 秒
- 边等边说："下载过程看左上角那行字，如果卡在 'Receiving objects' 别动，是网慢。"

#### 32:00 第 3 步：装依赖

**【动作】**
- 在终端敲 `uv sync`
- 故意停顿 2 秒让它开始下载

**【台词】**

> "这一行会下载整个项目的依赖。第一次会慢一点，1~3 分钟。下载完会自动建一个虚拟环境。"

#### 35:00 边等边讲

**【动作】**
- 走开让学生自己跑
- 边走边讲

**【台词·背景音】**

> "趁下载，我讲一下这套架构长啥样——你们不用记，听个响就行。"
>
> - "后端是 FastAPI，写 Python 的，特点是**快**。"
> - "AI 部分是 LangChain + OpenAI，主要就是调 GPT 的 API。"
> - "向量数据库是 ChromaDB，**你们要装的法律文档全在这里**。"
> - "前端是 Vue3，已经构建好放在 static 文件夹里，直接能用。"

#### 40:00 答疑与救场

**【动作】**
- 此时肯定有人报错
- 走过去看
- 大声说："报错不要慌，把最后三行念给我听。"

**【常见报错与应对】**

| 报错关键字 | 应对 |
|---|---|
| `python: command not found` | "去 python.org 下 3.12，重启终端" |
| `uv.lock lock failed` | "删 `uv.lock` 再 `uv sync`" |
| `Permission denied` (Mac) | "前面加 `sudo`" |
| 卡在 99% 不动 | "Ctrl+C 取消，重试一次" |

**【救场金句】**

> "环境报错是正常的，不报错才有鬼。我当年第一次装环境配了 3 天。"

#### 45:00 第 4 步：配 .env

**【动作】**
- 终端敲

**【台词 + 屏幕同步】**

```bash
cp .env.example .env
```

> "把示例配置复制成实际配置。然后用 VS Code 打开 .env 文件。"

#### 47:00 填 OPENAI_API_KEY

**【动作】**
- 走到白板前，把事先准备好的 key 撕开

**【台词】**

> "**重要：不要把我的 key 截图发朋友圈，不要发群，不要写到代码里提交。** 每个人过来看一眼，自己回去填。"

- 让一个学生上来把 key 写在黑板上
- 让学生照着填

**【互动】**
- 等 1 分钟
- 检查每组：填了举手

#### 50:00 切到 Part 4

## 附录 B：常见救场话术

### 场景 1：装环境报错率超过 30%

**【台词】**

> "你们之中应该有人装不上，没关系，环境问题就是看运气的。装不上的人举手，我们组对组帮扶——已经装好的组去帮装不上的组。**5 分钟后再来找我。**"

### 场景 2：API key 额度用完

**【台词】**

> "如果你看到 'insufficient_quota'，说明 key 没钱了。不要慌，截图发我，我给你换一个临时的。**不要自己充钱**。"

### 场景 3：时间超了

**【台词】**

**【台词】**

> "好问题。一句话：uv 比 pip 快 10 倍。我们这个项目依赖 100 多个包，pip 装要 5 分钟，uv 装 30 秒。多出来的时间你们可以多写 3 个 bug。"

### 场景 5：学生问"为什么 admin 密码是 123456"

**【台词】**

> "开发环境默认密码，方便调试。**生产环境绝对不能这么干**。我们 Day 4 讲安全的时候会讲为什么，以及怎么改。"
