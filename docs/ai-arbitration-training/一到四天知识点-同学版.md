# AI 劳动仲裁辅助系统 1-4 天知识点整理

> 给同学复习和预习使用。重点不是背代码，而是知道每天在解决什么问题、学到什么技术、最后要交付什么结果。

## Day 1：项目认知、环境搭建与 AI 基础

### 今天目标

把项目跑起来，知道这个系统解决什么业务问题，并理解 AI 问答的最小链路。

### 核心知识点

1. 项目定位
   - 本项目是一个 AI 劳动仲裁辅助系统。
   - 目标是辅助用户完成法律咨询、诉求分析、证据整理和仲裁文书草稿生成。
   - AI 只能辅助整理和生成建议，不能替代律师、仲裁员或最终法律判断。

2. 项目技术栈
   - 后端：Python、FastAPI、SQLAlchemy、Alembic、MySQL、Redis。
   - 前端：Vue 3、Vite、TypeScript、Element Plus。
   - AI：大模型、Prompt、RAG、Embedding、ChromaDB。
   - 包管理：uv。

3. 环境与启动
   - 会检查 Python、Git、uv、MySQL 等环境。
   - 会使用 `uv sync` 安装后端依赖。
   - 会使用 `uv run main.py run --env=dev` 启动后端。
   - 会打开 `/docs` 查看接口文档。

4. AI 基础概念
   - LLM：大语言模型，负责理解和生成文本。
   - Prompt：给模型的任务说明。
   - Embedding：把文本转换成向量，方便做语义检索。
   - RAG：先检索知识库，再把检索结果和问题一起交给模型回答。
   - ChromaDB：保存向量和文本片段，用于知识库检索。

5. 知识库基础流程
   - 创建知识库。
   - 上传法律文档。
   - 后端解析文档并切分文本。
   - 生成 Embedding。
   - 写入 ChromaDB。
   - 通过检索测试验证效果。

### 今天要会

- 能启动后端项目。
- 能打开接口文档。
- 能说清楚什么是 RAG。
- 能说明“用户问题 -> 知识库检索 -> Prompt -> 模型回答 -> 人工复核”的流程。
- 能创建知识库并上传文档做一次检索测试。

### 当天产出

- 项目能在本地启动。
- 依赖安装成功。
- 知识库上传和检索测试跑通。
- 能用自己的话解释 AI、Prompt、Embedding、RAG 的关系。

### 复习问题

## Day 2：RAG 检索、提示词工程与 AI 对话

### 今天目标

让 AI 回答更有依据、更稳定，并能通过普通接口或 WebSocket 完成对话。

### 核心知识点

1. RAG 检索质量
   - 检索输入：用户问题。
   - 检索输出：与问题相关的知识片段。
   - 检索质量会直接影响 AI 回答质量。
   - 法律场景要关注依据、条款、事实边界和风险提示。

2. 检索方式
   - 向量检索：适合语义相近的问题。
   - 关键词检索：适合法条编号、精确词、专有名词。
   - 混合检索：结合语义和关键词，提高召回效果。
   - 结果排序：把更相关、更可靠的内容排在前面。

3. Prompt 工程
   - 好的 Prompt 通常包含：角色、任务、输入、输出格式、约束条件、示例。
   - 法律类 Prompt 要强调不能编造事实，无法判断时要提示补充材料。
   - 输出可以约束为 Markdown、表格或 JSON。
   - Few-shot 示例可以让模型更稳定地按照指定格式回答。

4. AI 聊天接口
   - 普通问答接口适合一次性请求和响应。
   - WebSocket 适合流式输出，用户体验更像真实聊天。
   - 会话需要保存历史消息，便于查看详情和继续上下文。

5. 业务 Prompt 设计
   - 诉求分析：根据事实判断可能的仲裁请求。
   - 成功率评估：根据证据、事实和法律依据做风险判断。
   - 注意：这些能力可以先做 Prompt 或设计接口演示，不一定都已经完整后端落库。

### 今天要会

- 能解释检索输入和检索输出。
- 能写出一个结构清楚的法律咨询 Prompt。
- 能区分普通 HTTP 问答和 WebSocket 流式对话。
- 能通过 AI 对话页面或接口完成一次问答。
- 能说明为什么 Prompt 需要输出格式和边界约束。

### 当天产出

- 至少一套可用 Prompt 模板。
- AI 聊天接口或 WebSocket 对话跑通。
- 能演示一次带知识库依据的回答。
- 能设计诉求分析和成功率评估的输入输出。

### 复习问题

- RAG 检索结果不准时，AI 回答会发生什么？
- Prompt 中“角色、任务、输出格式、约束”分别有什么作用？
- WebSocket 为什么适合 AI 流式回答？
- 法律场景为什么要让 AI 标注“不确定”和“需补充材料”？

## Day 3：证据管理、文件解析与文书生成

### 今天目标

把业务从“会问答”推进到“能处理材料、分析证据、生成文书草稿”。

### 核心知识点

1. 文件上传
   - 前端选择文件，后端通过接口接收。
   - 后端需要保存文件，并记录文件元数据。
   - 上传后不能让用户一直等待，较慢任务可以放到后台处理。

2. 文件解析
   - PDF：提取合同、通知书、裁决书等文本。
   - Word：提取文档正文。
   - Excel：提取工资流水、考勤记录等表格数据。
   - 图片 OCR：可作为扩展，用于聊天截图、图片证据识别。

3. 后台任务
   - 上传文件后立即返回，后台继续解析、切片、索引。
   - 后台任务内部要自己处理异常和状态更新。
   - 长任务要考虑任务状态、失败原因和重新索引。

4. 证据分析
   - 证据类型：劳动合同、工资流水、解除通知、聊天记录、考勤记录等。
   - 证据价值：能证明什么事实。
   - 证据风险：是否缺页、是否模糊、是否缺少关键字段。
   - AI 可以辅助分类和总结，但需要人工复核。

5. 文书生成
   - 仲裁申请书通常包含：申请人、被申请人、仲裁请求、事实与理由、证据清单。
   - 可用模板引擎生成 HTML 或文书草稿。
   - 模板负责结构稳定，AI 负责根据事实补充内容。

6. 工程优化
   - Gzip 压缩：减少响应体积。
   - 数据库连接池：提升并发访问能力。
   - 统一异常处理：让接口错误更清楚、更一致。

### 今天要会

- 能说明文件上传到解析、索引的完整流程。
- 能说清楚 PDF、Excel、图片证据的处理差异。
- 能设计一个证据分析 Prompt。
- 能理解文书模板和 AI 生成内容的分工。
- 能做一次从材料上传到文书草稿的演示或设计说明。

### 当天产出

- 文档上传和知识库索引链路跑通。
- 至少能说明两类文件如何解析。
- 有一份证据分析结果或 Prompt 设计。
- 有一份仲裁申请书草稿模板或接口设计。
- 答辩 PPT 初稿。

### 复习问题

- 为什么文件上传后适合用后台任务处理？
- 证据分析时需要关注哪些维度？
- 文书生成为什么不能完全交给 AI 自由发挥？
- 模板引擎和 Prompt 分别解决什么问题？

## Day 4：安全认证、联调部署与项目答辩

### 今天目标

让系统具备基本安全保护，完成前后端联调，准备最终演示和答辩。

### 核心知识点

1. JWT 身份认证
   - 用户登录成功后，后端签发 Token。
   - 前端请求接口时携带 Token。
   - 后端验证 Token，确认用户身份和权限。
   - Token 不能随意泄露，过期时间要合理设置。

2. 权限与数据安全
   - 不是所有用户都能访问所有接口。
   - 后端要根据用户身份判断能否访问资源。
   - 敏感配置不能写死在代码里，例如数据库密码、API Key、SECRET_KEY。

3. 扩展安全知识
   - 密码安全：不能明文保存密码，要使用哈希。
   - 第三方登录：理解授权流程和风险点。
   - 接口限流：防止接口被恶意频繁请求。
   - 这些属于扩展了解，不一定要求全部现场完整实现。

4. 前后端联调
   - 检查接口地址、端口、跨域配置。
   - 检查 WebSocket 地址是否正确。
   - 检查文件上传、知识库检索、AI 对话是否能端到端跑通。

5. 部署基础
   - 开发环境和生产环境配置要分开。
   - 生产环境要检查数据库、Redis、模型接口、向量库目录。
   - Gunicorn、Nginx 等用于更稳定地运行服务。

6. 项目答辩
   - 讲清楚业务痛点：劳动者不知道怎么咨询、准备证据和写文书。
   - 讲清楚系统流程：咨询 -> 检索 -> 分析 -> 证据 -> 文书。
   - 讲清楚技术亮点：FastAPI、Vue、RAG、Prompt、WebSocket、知识库。
   - 讲清楚边界：AI 只辅助，结果需要人工复核。

### 今天要会

- 能解释 JWT 的登录和鉴权流程。
- 能说明 Token、SECRET_KEY、权限控制为什么重要。
- 能完成一次前后端端到端演示。
- 能准备项目 PPT 和演示台词。
- 能回答 RAG、Prompt、文件解析、安全认证相关问题。

### 当天产出

- 登录和 Token 鉴权链路说明清楚。
- 前后端核心功能联调通过。
- 项目演示流程稳定。
- 答辩 PPT 和讲稿完成。
- 常见答辩问题准备完成。

### 复习问题
