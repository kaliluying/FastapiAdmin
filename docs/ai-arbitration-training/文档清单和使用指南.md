## 🎯 文档使用指南

### 讲师使用

1. **备课准备**：
   - 先看 `完整四天授课执行脚本.md`，明确每天讲什么、做什么、验收什么
   - 再用 `courseware/index.html` 对照投屏页面，确认每页的话术和节奏
   - 需要解释代码细节时，再查对应的 Day1-Day4 演讲稿

2. **课堂教学**：
   - 以网页课件为主，不让学生一直跟着长演讲稿翻页
   - 每个阶段都要落到一个可见产出：能启动、能提问、能检索、能生成、能答辩
   - Day4 的第三方登录、密码安全、接口限流作为扩展讲解，不要求全部现场完整实现

3. **学生答疑**：
   - 使用"常见问题排查"章节快速定位问题
   - 引导学生查阅 uv 文档解决依赖问题

### 学生使用

1. **课前预习**：
   - 阅读 `AI仲裁辅助系统项目讲解文档.md` 的项目概览和运行部分
   - 了解项目背景和技术栈

2. **跟随实践**：
   - 跟随讲师网页课件和现场演示操作
   - 遇到问题查阅对应章节

3. **课后复习**：
   - 重新阅读演讲稿，巩固知识点
   - 完成练习，查阅讲解文档中的代码示例

### 项目开发者使用

1. **快速上手**：
   - 按照 `使用uv管理项目依赖.md` 安装环境
   - 参考讲解文档的"项目部署和运行"章节

2. **功能开发**：
   - 查阅"核心业务功能详解"了解业务逻辑
   - 参考"AI 核心技术深度解析"实现新功能

3. **故障排查**：
   - 使用"常见问题排查"快速定位
   - 查阅 uv 文档解决依赖冲突

## 📝 当前推荐使用顺序

1. **讲课前一天**：先看 `完整四天授课执行脚本.md`，确认当天动作和话术。
2. **正式讲课时**：打开 `courseware/index.html`，按页面顺序讲。
3. **讲到代码时**：切到当前骨架项目，围绕 `backend/app/plugin/module_ai/chat/*`、`backend/app/plugin/module_ai/knowledge/*` 和 `backend/app/core/*` 解释。
4. **学生卡住时**：回到 Day1-Day4 演讲稿查补充解释，不把长文档直接投给学生。
5. **答辩前**：用 `课程总纲-任务流程-功能阶段表.md` 和 `完整四天授课执行脚本.md` 统一验收口径。
