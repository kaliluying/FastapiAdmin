## 🎯 推荐的同步策略

由于历史记录不兼容，建议采用以下策略之一：

### 方案 1：手动选择性同步（推荐）

适合你已经删除了大量组件的情况。

```bash
# 1. 查看上游有哪些重要更新
git fetch origin
git log origin/master --oneline -20

# 2. 查看具体某个提交的改动
git show <commit-hash>

# 3. 选择性应用某个提交（cherry-pick）
git checkout custom-dev
git cherry-pick <commit-hash>

# 4. 如果有冲突，解决后继续
git add .
git cherry-pick --continue
```

### 方案 2：基于最新上游重新定制

如果上游改动太大，不如重新开始：

```bash
# 1. 基于最新上游创建新的定制分支
git checkout -b custom-dev-v2 origin/master

# 2. 参考你的删除记录，重新删除不需要的组件
# （可以对比 custom-dev 分支查看你删除了什么）
git diff custom-dev origin/master --name-status | grep "^D"

# 3. 删除组件后提交
git add .
git commit -m "chore: 移除不需要的组件（基于最新上游）"

# 4. 用新分支替换旧分支
git branch -D custom-dev
git branch -m custom-dev-v2 custom-dev
git push origin custom-dev --force
```

### 方案 3：查看文件级别的差异并手动更新

```bash
# 查看上游修改了哪些你保留的文件
git diff main origin/master --name-status | grep -v "^D"

# 对于重要的文件，手动查看差异
git diff main origin/master -- backend/app/core/security.py

# 如果需要，手动更新这些文件
```
